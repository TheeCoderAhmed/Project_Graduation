Repository source read, 2026-05-27.

README.md:
- Project name: DRAPO - Patient-Centric Healthcare Review Platform.
- Stack: Flutter 3.19+ / Dart 3.3+, provider package, Firebase Authentication, Firestore, Cloud Functions v2 Node.js.
- Codebase: lib/ contains about 8,500 Dart lines; verified count is 8,541 Dart lines across 52 Dart files.
- Data model includes users, providers, reviews, community_doctors, and community_reviews.
- Ranking formula: rankingScore = 0.4 * averageOverallRating + 0.6 * weightedQuestionnaireScore.
- AHP-inspired weights: staff communication 0.35, hygiene 0.25, service quality 0.25, waiting time 0.15.

PRODUCT.md and DESIGN.md:
- Audience: patients in Turkey and healthcare practitioners.
- Product purpose: search providers, leave post-visit reviews, complete structured questionnaires, view ranked provider listings.
- Brand personality: professional, modern corporate, minimalistic, clinical precision with human empathy, "Guided Confidence."
- Visual system: clinical blue, mint teal, cool neutrals, warm amber for ratings.

STUDY_GUIDE.md:
- Role split: patient, provider, admin.
- Patient flow: browse, search, community, reviews, profile.
- Provider flow: dashboard, own listing, reply to reviews, request practice changes.
- Admin flow: approve/reject practice changes.
- Community reviews: off-app doctors grouped by deterministic slug from name + hospital.

functions/src/aggregation.js:
- calculateProviderStats computes averageRating, avgQuestionnaireScore, and rankingScore server-side.
- round2 returns two-decimal metrics.
- AHP weights sum to 1.0.

firestore.rules:
- Users can read/write own document; admin can read.
- Signup allows patient/provider roles; admin cannot self-register.
- Provider score fields are locked from providers.
- Reviews use deterministic ID request.auth.uid + "_" + providerId, are patient-only, immutable except provider replies.
- Community reviews use deterministic user + communityDoctor ID, patient-only create, no edits.

lib/services/firestore_service.dart:
- getTopProviders orders by rankingScore.
- submitReview uses a transaction and deterministic review ID.
- submitCommunityReview uses a transaction to create/increment aggregated off-app doctor listing and write the review.
- requestPracticeChange writes pending fields; approve/reject applies or discards them through admin flow.
